# Portfolio Coach

게임 업계 취업 준비를 돕는 포트폴리오 코칭 서비스입니다.  
이력서, 자기소개서, 포트폴리오, GitHub 저장소, 직무 정보 기반으로 서류 피드백, 추천 공고, 면접 준비 자료를 한곳에서 확인할 수 있습니다.

- 서비스 주소: [https://yang-yun-seok.github.io/Portfolio-Coach/](https://yang-yun-seok.github.io/Portfolio-Coach/)
- 지원 직군: 기획, 프로그래밍, 아트

## 이 서비스로 할 수 있는 일

- 서류 피드백: 이력서와 자기소개서 문장을 직무 기준으로 점검
- 포트폴리오 리뷰: 구성, 강점, 보완점, 면접 설명 포인트 정리
- GitHub 분석: 프로그래밍 직군 기준으로 README, 구조, 설정, 테스트 신호 분석
- 추천 공고 매칭: 게임잡 공고 중 현재 프로필과 맞는 공고만 선별
- 면접 준비: 예상 질문, 답변 방향, 기본 준비 체크
- 직무 과제 평가: 기획, 프로그래밍, 아트 직군별 과제 연습
- 인성검사 시뮬레이션: 실제 기업 검사 흐름과 유사한 형태로 연습
- PDF 출력: 결과를 정리해서 보관하거나 제출용으로 활용

## 사용 순서

1. `정보 입력` 탭에서 이름, 직군, 세부 직무, 경력, 보유 역량을 입력합니다.
2. 필요한 경우 이력서, 자기소개서, 포트폴리오 PDF를 첨부합니다.
3. 프로그래밍 직군이라면 public GitHub 저장소 URL을 함께 입력합니다.
4. `AI 분석 시작` 버튼으로 기본 분석을 실행합니다.
5. 결과는 `서류 피드백`, `포트폴리오`, `면접 대비` 탭에서 확인합니다.
6. `추천 공고` 탭에서는 자동 매칭되지 않으며, 사용자가 `매칭하기` 버튼을 눌렀을 때만 AI 매칭이 실행됩니다.

## 추천 공고 동작 방식

- 게임잡 공고는 매일 00:00 KST 기준으로 자동 수집됩니다.
- 현재 수집 대상:
  - 게임개발(클라이언트)
  - 게임개발(모바일)
  - 게임기획
  - 게임운영
  - QA·테스트
- 현재 수집 경력 조건:
  - 신입
  - 1~3년
  - 경력무관
- 사용자 화면에서는 수동 크롤링 버튼이 노출되지 않습니다.
- 최근 반영일, 참고 공고 수, 마지막 성공 시각은 자동 배치 기준으로 갱신됩니다.

## 입력 팁

- 직무는 최대한 정확하게 선택하는 편이 좋습니다.
- 보유 역량은 추상적인 표현보다 실제 작업 단위로 적는 편이 좋습니다.
- 예시:
  - `Unity UI 구현`
  - `시스템 기획 문서 작성`
  - `2D 캐릭터 원화 제작`
  - `전투 밸런스 테이블 설계`
- PDF 첫 페이지에는 이름, 지원 직무, 프로젝트명, 본인 역할이 드러나게 정리하는 편이 좋습니다.

## GitHub 저장소 분석 안내

- 프로그래밍 직군에서 public GitHub 저장소 URL을 입력하면 추가 분석이 가능합니다.
- 현재 분석 항목:
  - README 정리 상태
  - 폴더 구조
  - 실행 스크립트
  - 설정 파일
  - 테스트 유무
  - 배포 및 협업 신호
- private 저장소는 직접 분석하지 않습니다.

## 저장소 구조

이 저장소에서 중요한 폴더만 보면 됩니다.

```text
.github/workflows/   GitHub Pages 배포, 일일 게임잡 크롤링
config/              모델 및 서버 설정
data/                회사 데이터, 게임잡 공고 데이터, 일별 공고 이력
lib/                 공고 정규화, 데이터 로더, 크롤러 공용 로직
prompts/             AI 분석 프롬프트
public/              정적 배포 자산, 공개 API JSON
scripts/             정적 API 생성, 스모크 테스트, 일일 크롤링 실행
server/              Express 라우트와 서비스 계층
src/                 React 프론트엔드
```

## 배포 구조

- Frontend: GitHub Pages
- Backend: Render Web Service
- AI Proxy: 서버 API
- Static Data: `public/api/*.json`

정적 페이지는 GitHub Pages에 배포되고, 서버 기능이 필요한 요청은 Render 백엔드가 처리합니다.

## 로컬 실행

```bash
npm install
npm run dev
```

- Frontend: `http://localhost:5173`
- Backend: `http://localhost:3002`

## 검증 명령어

```bash
npm run verify
```

이 명령은 아래를 순서대로 실행합니다.

- 정적 API 생성
- 프로덕션 빌드
- 스모크 테스트

## 자동 크롤링 이력

일일 크롤링 결과는 아래 두 경로에 누적됩니다.

- 원본 이력: `data/history/YYYY-MM-DD.json`
- 배포용 이력: `public/api/history/YYYY-MM-DD.json`

이력 목록 인덱스:

- `public/api/jobs-history-index.json`

## 운영 메모

- AI 분석은 서버 상태와 첨부 파일 크기에 따라 다소 시간이 걸릴 수 있습니다.
- 분석 중에는 새로고침이나 탭 종료를 하지 않는 편이 안전합니다.
- 추천 공고는 매일 자동 수집된 데이터 기준으로만 표시됩니다.
- 수동 크롤링 기능은 일반 사용자 화면에서 노출되지 않습니다.
